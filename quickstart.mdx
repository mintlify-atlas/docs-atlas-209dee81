GoMa'luum is a stateless proxy API for IIUM's i-Ma'luum student portal. It handles the session cookie lifecycle for you and returns clean JSON. All you need is your IIUM username and password.

<Note>
  The PASETO token returned by the login endpoint wraps your active IIUM session cookie. Treat it exactly like a password — do not log it, commit it to version control, or expose it client-side.
</Note>

## Base URL

```
https://api.quddus.my
```

A Swagger/Scalar reference is available at `https://api.quddus.my/api/reference`.

---

## Make your first API call

<Steps>
  <Step title="Log in and get your token">
    Send your IIUM credentials to the login endpoint. The response contains a PASETO token you will use for every subsequent request.

    <CodeGroup>
      ```bash curl
      curl -X POST https://api.quddus.my/api/auth/login \
        -H "Content-Type: application/json" \
        -d '{"username": "your_matric_number", "password": "your_password"}'
      ```

      ```javascript JavaScript
      const response = await fetch('https://api.quddus.my/api/auth/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          username: 'your_matric_number',
          password: 'your_password',
        }),
      });

      const { data } = await response.json();
      const token = data.token;
      ```

      ```python Python
      import requests

      resp = requests.post(
          'https://api.quddus.my/api/auth/login',
          json={'username': 'your_matric_number', 'password': 'your_password'},
      )

      token = resp.json()['data']['token']
      ```
    </CodeGroup>

    A successful response looks like this:

    ```json
    {
      "message": "Login successful! Please use the token in the Authorization header for future requests.",
      "data": {
        "token": "v4.public.eyJhdWQiOiJnb21hbHV1bSIs...",
        "username": "your_matric_number"
      }
    }
    ```

    Copy the value of `data.token` — you will need it in the next step.
  </Step>

  <Step title="Call a protected endpoint">
    Pass the token as a Bearer credential in the `Authorization` header. The `/api/profile` endpoint is a good first call to verify everything is working.

    ```bash
    curl https://api.quddus.my/api/profile \
      -H "Authorization: Bearer v4.public.eyJhdWQiOiJnb21hbHV1bSIs..."
    ```

    The server validates the PASETO token, extracts the session cookie, fetches your profile from i-Ma'luum, and returns the data as JSON.

    <Tip>
      The same `Authorization: Bearer <token>` pattern applies to every protected route — `/api/schedule`, `/api/result`, `/api/starpoint`, and others.
    </Tip>
  </Step>

  <Step title="Log out (optional)">
    Calling the logout endpoint invalidates your session on IIUM's CAS server. The PASETO token itself is stateless and will not be revoked server-side, but the underlying i-Ma'luum cookie it carries will be cleared.

    ```bash
    curl https://api.quddus.my/api/auth/logout \
      -H "Authorization: Bearer v4.public.eyJhdWQiOiJnb21hbHV1bSIs..."
    ```
  </Step>
</Steps>

## Enhanced security with API keys (optional)

By default, GoMa'luum encrypts your session data with PASETO. You can add a second encryption layer by generating your own API key and attaching it to every request via the `x-gomaluum-key` header. Tokens minted with your key can only be decrypted using that same key.

<Warning>
  A token generated with an API key cannot be used without that same key. If you lose the key, you must log in again to obtain a new token.
</Warning>

<Steps>
  <Step title="Generate an API key">
    This endpoint requires no authentication. Call it once per application or environment.

    ```bash
    curl -X POST https://api.quddus.my/api/key/generate
    ```

    ```json
    {
      "api_key": "gml_1703123456_a1b2c3d4e5f6789012345678901234567890abcd",
      "message": "API key generated successfully. Include this key in the 'x-gomaluum-key' header for enhanced security.",
      "created_at": "now"
    }
    ```

    Store the `api_key` value somewhere secure — an environment variable or a secrets manager. Never commit it to source control.
  </Step>

  <Step title="Log in with your API key">
    Include the `x-gomaluum-key` header alongside your credentials. The resulting PASETO token will be bound to this key.

    ```bash
    curl -X POST https://api.quddus.my/api/auth/login \
      -H "Content-Type: application/json" \
      -H "x-gomaluum-key: gml_1703123456_a1b2c3d4e5f6789012345678901234567890abcd" \
      -d '{"username": "your_matric_number", "password": "your_password"}'
    ```
  </Step>

  <Step title="Include the API key in every subsequent request">
    All protected endpoints must receive both the `Authorization` header and the `x-gomaluum-key` header.

    ```bash
    curl https://api.quddus.my/api/profile \
      -H "Authorization: Bearer v4.public.eyJhdWQiOiJnb21hbHV1bSIs..." \
      -H "x-gomaluum-key: gml_1703123456_a1b2c3d4e5f6789012345678901234567890abcd"
    ```
  </Step>
</Steps>
